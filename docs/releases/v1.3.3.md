# MUG v1.3.3 Release Notes

Release date: 2026-05-12

## Summary

MUG v1.3.3 completes the global phase synchronization workflow by integrating the `(V) x (I)` graph into the same behavior already used by Tensão, Corrente, DHT Tensão, and DHT Corrente. It also improves the update workflow so the user is sent directly to the installer asset and the application closes before installation.

The release preserves the existing ETL architecture, benchmark/profiling infrastructure, lazy loading, zoom/rebuild behavior, PDF pipeline, PyInstaller packaging, and Inno Setup installer flow.

## Highlights

- Full integration of `(V) x (I)` into the global phase synchronization ecosystem.
- Bidirectional synchronization across:
  - Tensão
  - Corrente
  - DHT Tensão
  - DHT Corrente
  - `(V) x (I)`
- Incremental propagation using `Plotly.restyle()`.
- Existing zoom/rebuild behavior preserved.
- Dynamic Máx/Mín recalculation using only visible phases.
- Phase-aware extreme labels, for example:
  - `386,25 V (ST)`
  - `363,05 A (S)`
- Correct separation of combined graph extreme groups:
  - Máx/Mín (V)
  - Máx/Mín (A)
- PDF export now respects visible phases, recalculated extremes, and phase-aware labels for `(V) x (I)`.

## Updater Improvements

The update workflow now opens the installer asset directly instead of the general GitHub release page.

The URL is built dynamically from the detected latest version:

```text
https://github.com/pancotto/MUG/releases/download/<versão>/MUG_Setup_<versão>.exe
```

Example:

```text
https://github.com/pancotto/MUG/releases/download/v1.3.3/MUG_Setup_v1.3.3.exe
```

When the user clicks `BAIXAR`, the application now:

1. Opens the direct installer download URL.
2. Falls back to the release page if opening the direct URL fails.
3. Closes the main window.
4. Gracefully quits `QApplication` before installation.

No internal downloader was added.

## Compatibility

- Primata XLSX remains supported.
- Primata TXT remains supported.
- Embrasul TXT remains supported.
- Existing ETL behavior is preserved.
- Existing benchmark/profiling infrastructure is preserved.
- Lazy loading of PDF exporter and update checker is preserved.
- PyInstaller onedir build remains the supported executable packaging flow.
- Inno Setup remains the supported Windows installer flow.
