# MUG v1.3.1 Release Notes

Release date: 2026-05-11

## Summary

MUG v1.3.1 consolidates profiling, benchmark automation, startup improvements,
and low-risk ETL optimizations. The release preserves the current application
architecture, visual behavior, graph generation flow, PDF export flow, and
PyInstaller distribution model.

## Highlights

- Added profiling for ETL, graph generation, rebuild/zoom, PDF export, per-graph
  export timing, and approximate memory reporting.
- Added automatic benchmark infrastructure with JSON/Markdown outputs, logs, and
  historical comparison support.
- Added project architecture documentation and agent instructions for safe future
  development.
- Lazy-loaded PDF exporter dependencies so PDF/Kaleido code is loaded only at
  export time.
- Lazy-loaded the update checker so update dependencies are loaded only when
  update verification runs.
- Added vectorized datetime fast path for Primata TXT with automatic fallback.
- Added dtype preservation/reuse in ETL to reduce redundant numeric conversion,
  especially for Primata XLSX.
- Improved stability and benchmark reproducibility without changing graph output
  semantics.

## Compatibility

- Primata XLSX remains supported.
- Primata TXT remains supported.
- Embrasul TXT remains supported.
- Existing `ProcessedData` flow is preserved.
- Existing graph behavior is preserved.
- Existing PDF export behavior is preserved.
- PyInstaller onedir build remains the supported executable packaging flow.
- Inno Setup remains the supported Windows installer flow.

## Benchmark Policy

Generated benchmark artifacts are not versioned:

- `docs/benchmarks/runs/`
- `docs/benchmarks/logs/`
- `docs/benchmarks/latest.*`
- `docs/benchmarks/artifacts/`

Official release benchmark references are stored in:

- `docs/benchmarks/releases/`

Real benchmark datasets under `benchmarks/datasets/` must remain outside Git.
Only `.gitkeep` and documentation for the dataset directory should be versioned.
